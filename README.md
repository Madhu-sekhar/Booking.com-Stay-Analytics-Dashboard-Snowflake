# Booking.com-Stay-Analytics-Dashboard-Snowflake
Business Requirements Document – Hotel Analytics Dashboard

Objective:
To build a Hotel Analytics Dashboard that provides clear visibility into hotel bookings, revenue, and booking status using cleaned and structured data.

## Key Requirements:
Clean raw booking dataset
Fix invalid or inconsistent dates
Validate booking status values
Ensure accurate revenue calculations

## KPIs to Display:
Total Revenue
Total Bookings

## Functional Requirements:
Clean raw dataset (fix dates, remove invalid values)
Create monthly booking and revenue metrics

## Generate analytics for:

Monthly bookings
Monthly revenue
Top revenue-generating cities
Bookings by status (bar chart)
Expose KPIs on the dashboard

## Deliverables:

Accurate monthly metrics
Correct KPIs
Dashboard that is easy to interpret
No data quality issues in the final output

Built a hotel analytics dashboard using Snowflake by loading CSV data through internal stages and file formats, transforming data with SQL, and implementing a Bronze–Silver–Gold data model. Created Gold-layer aggregates for bookings and revenue and visualized KPIs using Snowsight.
Future scope includes automated ingestion, incremental loads, role-based access control, and predictive analytics.
